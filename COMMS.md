# Dungeon Coverage - Communications

## LinkedIn Post

---

Built a little game this weekend — it teaches code coverage by turning functions into dungeons.

Each function becomes a dungeon map. if/else branches become forking paths, loops become corridors. You drag test inputs into parameter slots, hit run, and a little hero walks through the code path your inputs trigger. Every statement you hit drops a gem. Miss a branch? There's a room you can't reach yet.

100% coverage = level complete.

There are six levels that go from simple if/else all the way to async functions with mocks. It's silly but I think it actually makes coverage click for people who haven't thought about it before.

Built with PIXI.js and powered by [maineffectjs](https://github.com/arvindrajnaidu/maineffect).

Play it: https://arvindrajnaidu.github.io/dungeon-coverage/
Source: https://github.com/arvindrajnaidu/dungeon-coverage
